# 비지도학습 기반 맥주 소비자 선호 세그먼트 분석

**프로젝트 분류**: STAT 437(통계학) 학기말 프로젝트 / 클러스터링(비지도학습)
**데이터**: Kaggle "Beer Profile and Ratings" — 3,197개 맥주, 934개 양조장, 25개 변수
**핵심 기법**: t-SNE 군집가능성 검증, DBSCAN, HAC(Complete Linkage), K-Means, 다중 평가지표 교차검증

---

## 1. 서론

### 1.1. 연구 동기

비지도학습은 라벨이 없는 데이터에서 숨은 패턴과 관계를 발견하는 기법으로, 고객 선호와 행동을 파악하는 비즈니스 전략 도구로 널리 쓰인다. 본 프로젝트는 이를 활용해 전 세계에서 가장 많이 소비되는 주류 제품인 **맥주**의 소비자 세분화를 분석한다.

맥주의 알코올 스펙과 맛 프로파일 변수만으로 내재적(inherent) 군집 구조가 존재하는지 확인하고, 각 군집이 어떤 특성을 가지며 무엇이 **고평점 맥주**를 만드는지 규명하는 것이 목표다.

**연구 목표**
1. 맥주 데이터의 자연 발생적 주요 군집 식별
2. 주요 군집 내부의 서브 군집 구조 식별
3. 이상치/노이즈 식별
4. 군집을 특성화하고 고평점 맥주의 속성 분석
5. 군집 간 유사도/비유사도와 그 구분 이유 이해
6. 맥주 특성 변수 간 연관성 파악(추천 시스템의 기반)

### 1.2. 활용 방안

분석 결과는 양조장 영업 담당자나 맥주 회사 경영진에게 유용하다. 예를 들어 "단맛 + 과일향" 속성에 고평점이 몰린 군집이 발견되면, 양조사는 그 세그먼트를 겨냥한 신제품을 만들거나 경쟁 제품을 개발할 수 있다. 장기적으로는 고객 만족과 브랜드 충성도 강화로 시장 내 입지를 다지는 데 기여한다.

### 1.3. 잠재적 한계

모든 내재 군집을 식별하지 못하면 의미 있는 맥주 특성 구분에 실패할 수 있다. 또한 서브 군집이 많은 데이터에서는 군집의 과분할(불필요한 단일 관측치 군집 양산)이나 과소분할(서로 다른 두 군집을 잘못 병합)이 발생해 군집 특성이 왜곡될 수 있다.

### 1.4. 분석 절차

원본 데이터 탐색 → 결측치/이상치 점검 및 스케일링 → t-SNE로 군집가능성 시각화 → 데이터 특성에 맞는 알고리즘 선택 → t-SNE 대조·분리도·연구목표 부합성으로 알고리즘 비교 → 최종 알고리즘 선정 및 결론·제언.

## 2. 데이터셋

### 2.1. 출처와 구성

Kaggle 사용자 ruthgn이 "Beer Tasting Profiles"와 "1.5 Million Beer Reviews" 두 데이터셋을 통합해 만든 데이터로, 3,197개 고유 맥주와 934개 양조장의 시음 프로파일·소비자 리뷰를 담는다. 통합 과정에서 FuzzyWuzzy로 이름 매칭이 안 된 맥주는 제외되어 일부 누락이 있으나, 전반적 패턴 분석에는 충분한 대표성을 갖는다.

### 2.2. 변수 구성

분석에는 수치형 변수만 사용하고, 과도하게 세분화되는 범주형 변수(Description, Brewery 934종, Style 111종, Name 3,197종)는 제외했다.

| 그룹 | 변수 |
| :--- | :--- |
| 스펙 | ABV(도수), Min IBU, Max IBU |
| 마우스필 | Astringency(떫음), Body(바디감), Alcohol |
| 맛 | Bitter, Sweet, Sour, Salty |
| 향·풍미 | Fruits, Hoppy, Spices, Malty |
| 리뷰 | review_aroma, appearance, palate, taste, overall, number_of_reviews |

> 참고: 시음 프로파일 변수는 맥주당 최대 25개 리뷰에서 추출한 **단어 빈도(word count)**로, 리뷰어가 언급한 특정 묘사어의 등장 정도를 정량화한 값이다.

## 3. 데이터 정제 및 탐색

### 3.1. 결측치 및 스케일링

전 변수 결측치 없음(모든 컬럼 3,197개 완전). 스펙·프로파일 변수는 값의 범위 차이가 커서(IBU 0~100 등) **StandardScaler로 표준화**해 특정 변수가 군집 구조를 지배하지 않도록 했다. 리뷰 변수(1~5 척도)는 이미 스케일이 일정해 표준화하지 않았다.

### 3.2. 핵심 설계 결정

**클러스터링은 스펙·프로파일 변수만 사용하고, 리뷰 변수는 군집 생성에서 제외**했다. 이는 인기도가 아닌 **제품 고유 특성**에 기반한 세그먼트를 도출하기 위함이며, 리뷰는 이후 군집 해석 단계에서만 활용했다. 거리 척도는 유클리드 거리를 사용했다.

## 4. t-SNE 군집가능성 검증

### 4.1. 방법

perplexity 6종(5, 10, 20, 30, 40, 50) × random state 2종(33, 437)으로 t-SNE를 그려 군집가능성을 시각적으로 검증했다.

### 4.2. 결과

- 데이터는 무작위 구름이 아니라 **밀집 영역, 분리된 소그룹, 영역 간 간격**을 보여 군집 가능성이 확인됐다.
- perplexity 30~40에서 구조가 가장 뚜렷해, **perplexity=30, random_state=33**을 대표 플롯으로 선정했다.
- **대략 4~6개의 주요 군집**이 추정되며, 하나의 큰 중심 덩어리 + 주변의 작은 특화 그룹 형태다.
- 군집 형태는 구형이 아니라 **불규칙·길쭉·부분 연결형**이라, K-Means 같은 중심 기반 가정에 완전히 부합하지 않고 밀도 기반 해석이 더 적절할 수 있음을 시사했다.
- 군집 크기는 불균형하며, 중심부에서 영역 간 **점진적 전이(연속체)**가 관찰됐다.

### 4.3. 속성별 구조 연관성

대표 t-SNE 플롯을 각 속성으로 색칠해 어떤 변수가 구조를 만드는지 확인했다.

- **IBU·Bitter·Hoppy**: 가장 명확한 연관. 좌측·좌하단에 고값 집중 → 쓴맛/홉 강도가 주요 분리 축.
- **Sour·Fruits**: 우측 그룹에 고값 집중 → 신맛/과일 중심 프로파일.
- **Malty·Body·Sweet**: 좌하단·중하단에 고값 → 풀바디·몰트 중심 맥주.
- **Alcohol·ABV**: 중하단에 국소적 고값 → 고도수 맥주의 작은 영역 식별.
- **Astringency·Spices**: 우측·우하단에 국소적 연관.
- **Salty**: 전체 구조와 거의 무관(소수 관측치만 고값).

## 5. 클러스터링 알고리즘 적용

서로 다른 원리를 대표하는 세 알고리즘을 적용하고, 각각 t-SNE 대조·실루엣 점수·cluster-sorted similarity matrix로 평가했다.

### 5.1. 알고리즘 1 — DBSCAN (밀도 기반)

**파라미터 탐색**: k-distance plot(n=3,5,8,10)으로 eps 후보(약 1.8~2.6)를 잡고, eps × min_samples 격자 탐색을 수행했다. 작은 eps는 군집이 많지만 노이즈 과다, 큰 eps는 노이즈는 줄지만 1~2개 거대 군집으로 병합되는 트레이드오프가 나타났다. 최고 실루엣은 매우 작은 eps에서 나왔으나 대부분을 노이즈로 분류해 비실용적이었다.

**최종 선정**: **eps=1.8, min_samples=8 → 4개 군집 + 513개(16%) 노이즈**. t-SNE 대조와 similarity matrix에서 가장 균형 잡힌 결과를 보였다.

**노이즈 해석**: 513개 노이즈는 오류가 아니라 American Barleywine 등 극단적 조합을 가진 **니치 스타일**로, 별도 그룹으로 보존하는 것이 타당하다.

**리뷰 분석**: Cluster 2가 평점·리뷰 수 모두 최고(프리미엄/인기 세그먼트), Cluster 1이 최저. 노이즈 맥주도 리뷰 점수는 양호했다.

### 5.2. 알고리즘 2 — HAC Complete Linkage (계층적)

**원본 데이터의 문제**: 덴드로그램이 1개 거대 군집 + 소수 이상치 형태로, 해석이 어려웠다. Complete Linkage는 두 그룹 간 가장 먼 쌍으로 병합 높이가 결정되어, 소수 극단치가 덴드로그램을 늘려 주류 맥주 구조를 가리는 약점이 있다.

**DBSCAN 노이즈를 입력으로 사용한 근거**: HAC가 정제할 데이터를 HAC로 정제하면 순환논리가 된다. 따라서 **밀도 기반(DBSCAN)과 연결 기반(HAC)이라는 서로 다른 원리를 교차 사용**해, 한 방법의 강점으로 다른 방법의 약점을 보완했다. DBSCAN이 식별한 513개 노이즈를 제거 후 HAC를 재적용했고, 이후 노이즈를 cluster -1로 다시 포함해 어떤 관측치도 버리지 않았다.

**노이즈 제거 후 결과**: 덴드로그램이 6~7개 군집 구조로 개선됐다. 실루엣은 k=2에서 최고(0.172)였으나, 이는 단순 분할 선호 경향 때문이며 t-SNE·similarity matrix 종합 판단으로 **k=7을 최종 선정**(과분할 회피 + 균형 잡힌 크기 + 명확한 대각 블록).

**군집 특성 분석**(스펙/프로파일/리뷰 3개 섹션, 박스플롯 기반):

| 군집 | 핵심 특성 | 리뷰 |
| :--- | :--- | :--- |
| Cluster 1 | 최고 홉·쓴맛·IBU·도수, 중간 풍미 | **최고 평점** |
| Cluster 3 | 모든 지표 최저(라이트) | 최저 평점, 호불호 큼 |
| Cluster 0 | 높은 쓴맛/홉, 그러나 단조로운 풍미 | 두 번째로 낮음 |
| Cluster 2,4,5,6 | 각각 과일·쓴맛·향신료·단맛에 특화 | 높은 평점 |
| Cluster -1(노이즈) | 모든 속성에서 다양·극단적 | 두 번째로 높음(니치) |

> 핵심 통찰: Cluster 0과 1은 쓴맛/홉이 비슷한데 평점은 크게 차이 났다. **쓴맛만으로는 부족하며, 이를 받쳐주는 보완 풍미가 있어야 고평점**으로 이어진다.

### 5.3. 알고리즘 3 — K-Means (중심 기반)

**파라미터 탐색**: 엘보 플롯에서 k=6~8 구간이 후보, 실루엣 점수는 k=8에서 최고. t-SNE 대조에서 k=5,6이 분리된 영역과 잘 맞았다.

**최종 선정**: t-SNE 대조와 해석성을 종합해 **k=6** 선정.

**리뷰 분석**: Cluster 1이 평점·리뷰 수 모두 최강(프리미엄), Cluster 5가 최저. 평점을 군집 생성에 쓰지 않았음에도 세그먼트별 평점 차이가 뚜렷해, 제품 특성 기반 군집이 소비자 평가에도 반영됨을 확인했다.

## 6. 방법 선택 및 논의

### 6.1. 알고리즘별 결과 요약

| 알고리즘 | 최종 설정 | 군집 수 | 노이즈 | 특징 |
| :--- | :--- | :--- | :--- | :--- |
| DBSCAN | eps=1.8, min_samples=8 | 4 | 513개(16%) | 니치 스타일을 노이즈로 분리 |
| HAC complete | k=7(노이즈 제거 후) | 7 | 사전 제거 | 대각 블록 가장 뚜렷, 풍부한 세부 구조 |
| K-Means | k=6 | 6 | 없음 | 가장 깔끔한 분할, 실루엣 최고 |

### 6.2. 세 기준에 따른 비교

- **내재 구조 복원**: 세 방법 모두 큰 그림 일치. K-Means·HAC가 t-SNE 분리 영역과 조금 더 깔끔하게 일치.
- **이상치 처리**: DBSCAN만 니치 맥주를 노이즈로 자연 분리(연구 목표에 직접 부합). K-Means는 강제 배정으로 중심 왜곡.
- **해석성·분리도**: K-Means가 가장 깔끔, HAC(k=7)가 가장 풍부한 세부 구조.

### 6.3. 세 방법이 다른 답을 내는 이유

작은 k에서 실루엣이 높은 것은 이상치가 군집 간 거리를 부풀린 착시다. 노이즈 제거 시 실루엣이 떨어지며 더 의미 있는 k=7이 드러난다. 맥주 프로파일은 쓴맛·단맛·몰트·도수를 따라 **연속적으로 변하는 스펙트럼**이라, 중심·연결 기반 방법은 이를 편의상 자르고 DBSCAN은 밀집 부분만 분리한다. 어느 방법도 틀리지 않았으며 조금씩 다른 질문에 답하는 것이다.

### 6.4. 최종 결정

- **DBSCAN(eps=1.8, min_samples=8) — 주방법**
- **HAC Complete Linkage(k=7) — 보완 방법**
- **K-Means(k=6) — 견고성 검증용**

DBSCAN이 먼저 니치/이상치를 분리하고, HAC가 남은 주류 맥주를 7개 세그먼트로 세분화한다. K-Means가 HAC 핵심 세그먼트와 강하게 일치한다는 점은 그 그룹들이 실제로 존재함을 뒷받침한다.

## 7. 결론

### 7.1. 핵심 발견

1. **군집은 존재하지만 연속체 형태다.** t-SNE 전반에서 큰 연결 핵심 + 작은 특화 그룹(4~6개 영역)이 일관되게 나타났고, 영역 간 전이가 점진적이다.
2. **구조를 만드는 것은 해석 가능한 맛 축이다.** 쓴맛/홉/IBU, 신맛/과일, 바디/몰트/단맛이 주축이며 짠맛은 거의 무관하다.
3. **니치/이상치 층이 분명히 존재한다.** DBSCAN이 약 16%를 노이즈로 분류했고, 이는 특이 스타일이며 리뷰도 양호했다.
4. **프로파일 군집이 소비자 선호와 연결된다.** 균형 잡히고 홉 향이 살아있는 맥주가 최고 평점을, 밋밋한 저강도 프로파일이 최저 평점을 받았다.

### 7.2. 연구 목표 달성 여부

| 목표 | 달성 |
| :--- | :--- |
| 주요 내재 군집 식별 | 달성 |
| 서브 군집 구조 식별 | 달성(HAC k=7) |
| 이상치/노이즈 식별 | 달성(DBSCAN 16%) |
| 군집 특성화·고평점 요인 | 달성(균형·복합성이 관건) |
| 유사도 기반 추천 | 부분 달성(기반 마련, 향후 과제) |

### 7.3. 비즈니스 시사점

가장 보상받는 영역은 **균형 잡히고 풍미가 복합적인 맥주**이지 최대한 쓰거나 독한 맥주가 아니다. 밋밋한 저평점 세그먼트는 포화/미차별 공간이며, 니치 층은 작지만 충성도 높은 고객을 노리는 의도적 특화 포지셔닝이다.

### 7.4. 한계 및 향후 과제

- **한계**: 표준화된 수치형 변수에 유클리드 거리만 사용, 스타일·양조장 등 범주형 맥락 제외. 프로파일은 리뷰 단어 빈도라 리뷰어 편향 내포. 연속적 구조로 "정답" k가 유일하지 않음.
- **향후 과제**: (a) 프로파일 유사도 기반 추천 시스템 구축, (b) 가변 밀도에 강한 GMM/HDBSCAN 테스트, (c) 스타일 라벨을 검증 기준으로 활용해 발견 군집과 기존 스타일의 일치도 확인.

## 8. 한 줄 요약

> 맥주 데이터는 뚜렷한 군집이 아니라 연속적인 맛의 스펙트럼이며, 이 특성 때문에 이상치 식별에 강한 **DBSCAN을 주방법**, 세부 세그먼트 해석에 강한 **HAC(k=7)를 보완**으로 결합했다. 분석 결과 고평점을 가르는 것은 맛의 강도가 아니라 **균형과 풍미의 복합성**이었다.
